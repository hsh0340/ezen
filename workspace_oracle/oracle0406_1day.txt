Oracle 사용 방법 2가지
1. sqlplus를 사용하는 방법 -> dos 화면, 명령프롬프트 화면, command 화면
-> text 환경

2. tool을 사용하는 방법
-> gui 환경

1. 데이터베이스 기본 개념
Datum - 단수형, 자료 1개
Data - 복수형, 자료들
DB(database) - data의 모음, 집합
DBMS(DataBase Management System) - 데이터베이스 관리 시스템
RDBMS(Relational DataBAse Management System) - 관계형 데이터베이스 관리 시스템
- 모든 데이터를 테이블 형태로 만들고, 관리하는 DBMS
Relation(table) - RDBMS에서 데이터를 저장하고 관리하는 테이블

2. RDBMS 제품
- Oracle, MySQL, MariaDB, MS-SQL, PostgreSQL...

3. SQL의 기본 개념과 종류
SQL(Structured Query Language) - 구조화된 질의 언어, RDBMS에서 사용하는 언어
(1) DCL(Data Control Language), 데이터 제어어
- 데이터베이스를 제어하고 관리하는 용도로 사용하는 언어, 데이터베이스 관리자가 주로 사용
- grant, revoke ...
(2) DDL(Data Definition Language), 데이터 정의어
- 데이터베이스를 정의하는 용도로 사용하는 언어
- create, alter, drop ...
(3) DML(Data Manipulation Language), 데이터 조작어
- 데이터베이스에 데이터를 삽입, 수정, 삭제하는 용도로 사용하는 언어. 개발자가 주로 사용.
- insert, update, delete, select ...

4. RDBMS의 권한
- 관리자 권한: sys, system
- 사용자 권한: 직접 만들어서 사용

5. 관리자 접속
sqlplus system/1234

6. 사용자 계정(account) 생성과 권한 부여
계정 생성 : create user ezen01 identified by 1234;
권한 부여 : grant connect, resources, dba to ezen01;

7. 사용자 접속
sqlplus ezen01/1234

8. 사용자 확인
show user;

9. 사용자 삭제
관리자 계정에서
drop user ezen02;

< employee, department, salgrade 테이블 정리>
1. employee - 사원의 정보를 가진 테이블
- eno, ename, job, manager, hiredate, salary, commission, don의 8개의 필드(열)

2. department - 부서의 정보를 가진 테이블
- dno, dname, loc의 3개의 필드(열)

3. salgrade - 급여의 등급과 하한가와 상한가의 정보를 가진 테이블
- grade, losal, hisal

10. 테이블의 정보를 확인하는 명령문
desc(ride) employee; -- 테이블의 구조를 확인하는 명령
select * from employee; -- 테이블의 데이터를 확인하는 명령

11. 자신의 계정에서 생성된 테이블을 확인하는 명령문
select table_name from user_tables;

12. 학습용 계정 hr 사용 방법
관리자 계정에서 
alter user hr account unlock; -- 계정의 잠금 해제
alter user hr identified by 1234; -- 계정의 비밀번호 변경
exit;
